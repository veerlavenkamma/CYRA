# Agent Spec — CYRA (Cyber Aura) Multi-Agent AI Security System

## Overview

CYRA (Cyber Aura) is a proactive cybersecurity concierge AI agent built on Google ADK
with the Gemini 2.5 Flash model. It monitors incoming chat logs, SMS texts, emails, and
call transcripts for cyber threats without seizing device control. CYRA operates via an
internal 3-step Multi-Agent / Routing Loop:

1. **Agent 1 (Parser/Extractor)**: `scan_text_stream` — isolates domains, TLD signals,
   credential keywords, and phishing patterns.
2. **Agent 2 (Threat Evaluator/Intel)**: `decrypt_link_redirects` + `context_privacy_guard`
   — inspects URLs for hidden malicious destinations, detects data-extraction attempts.
3. **Agent 3 (User Notifier)**: `analyze_call_transcript` + structured markdown alert
   — generates eye-catching, structured CYRA WARNING ALERT notifications.

## Example Use Cases

| Input | Expected Output |
|-------|----------------|
| SMS with `win-lottery-now.top` URL | ⚠️ CYRA WARNING ALERT — HIGH RISK (malicious domain + suspicious TLD) |
| "Enter your OTP to verify account" | ⚠️ CYRA WARNING ALERT — HIGH RISK (credential harvesting) |
| IRS arrest warrant call transcript | ⚠️ CYRA WARNING ALERT — HIGH RISK (government impersonation scam) |
| Friendly casual message from a friend | ✅ System Protected. Status: Safe. |
| Corporate follow-up email | ✅ System Protected. Status: Safe. |
| `bit.ly/kyc-verify` shortened URL | ⚠️ CYRA WARNING ALERT — HIGH RISK (URL shortener + hidden destination) |

## Tools Required

| Tool | Purpose | Auth |
|------|---------|------|
| `scan_text_stream` | Parse text for TLD signals, credential keywords, phishing patterns | None (local logic) |
| `decrypt_link_redirects` | Analyse URLs for redirect abuse, typo-squatting, shorteners | None (structural analysis) |
| `context_privacy_guard` | Detect financial/identity data extraction attempts | None (local logic) |
| `analyze_call_transcript` | Parse transcripts for vishing, impersonation, deepfake scams | None (local logic) |

## Constraints & Safety Rules

- NEVER provide hacking tools, exploit code, or pen-testing assistance
- NEVER ask users to share their personal data with CYRA itself
- ALWAYS call `scan_text_stream` before giving any verdict
- NEVER dismiss a concern without running appropriate analysis tools
- Advisory only — no device control or autonomous action

## Success Criteria

- `scan_text_stream` correctly detects 95%+ of test phishing messages
- `decrypt_link_redirects` flags all malicious domain DB entries and URL shorteners
- `context_privacy_guard` triggers on all credential-harvesting messages
- `analyze_call_transcript` catches government impersonation and tech support scams
- Clean/safe messages receive "✅ System Protected. Status: Safe." (< 5% false positive rate)
- Alert format compliance: all threat responses use the structured CYRA WARNING ALERT format
